# Runtime Plugin Registration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add runtime JS plugin loading and IR data access to the frontend_tk_rs CLI.

**Architecture:** A new NAPI function `get_ir` bridges Rust IR generation to JS plugins. JS plugins are loaded via `import()` after Commander parses `--plugin` args, registered with `cli.use()`. Only the JS CLI path (Commander) is affected.

**Tech Stack:** Rust/NAPI-RS (binding), TypeScript/Commander.js (CLI), ava (NAPI tests), vitest (JS tests)

**Spec:** `docs/spec/runtime-plugin-registration-design.md`

## Task 1: Add `get_ir` NAPI function (Rust)

**Files:**
- Modify: `crates/node_binding/src/lib.rs`

**Context:** The existing `run_cli` function in `lib.rs` already parses OpenAPI from a file path. The new `get_ir` function reuses the same file-reading + parsing logic, but calls `build_ir_snapshot_json` and returns the result as `serde_json::Value` instead of executing a command.

- [ ] **Step 1: Add `get_ir` function to `lib.rs`**

Add the following function after the existing `run_cli` function (after line 63):

```rust
#[napi]
pub fn get_ir(input_path: String) -> napi::Result<serde_json::Value> {
    let path = Path::new(&input_path);
    let abs_path = if path.is_absolute() {
        path.to_path_buf()
    } else {
        current_dir().unwrap().join(input_path)
    };
    let text = std::fs::read_to_string(&abs_path)
        .map_err(|err| Error::from_reason(format!("Failed to read OpenAPI file: {}", err)))?;
    let open_api = OpenAPIObject::from_str(&text)
        .map_err(|err| Error::from_reason(format!("OpenAPI parse error: {}", err)))?;
    let json_str = swagger_gen::pipeline::build_ir_snapshot_json(&open_api)
        .map_err(|err| Error::from_reason(err))?;
    let value: serde_json::Value = serde_json::from_str(&json_str)
        .map_err(|err| Error::from_reason(format!("JSON deserialization error: {}", err)))?;
    Ok(value)
}
```

Note: The necessary imports (`Path`, `current_dir`, `OpenAPIObject::from_str`, `Error`) are already present at the top of `lib.rs`. Only `swagger_gen::pipeline::build_ir_snapshot_json` needs to be in scope, which is available since `swagger_gen` is a dependency in `Cargo.toml`.

## Task 5: Plugin loading in `CliImpl.run()`

**Files:**
- Modify: `packages/frontend-tk-core/src/cli.ts`

**Context:** Currently `--plugin` is registered as a global option (`-p, --plugin <paths...>` in `cli.ts:86`) but not consumed anywhere. We add plugin loading logic inside `run()`, between Commander's `parseAsync` and command execution.

The challenge: Commander's `parseAsync` both parses args AND executes the matched command's action handler in a single call. To load plugins before command execution, we need to pre-parse global options first, load plugins, then re-parse for command execution.

The approach: Use `program.opts()` after a preliminary parse. Since `parseAsync` with `.exitOverride()` throws for help/version, we handle that. For the command execution case, we split into two phases:

1. Pre-parse to get `--plugin` and `--input` values
2. Load plugins via `cli.use()`
3. Let Commander execute the matched command (which now includes plugin commands)

- [ ] **Step 1: Add imports at the top of `cli.ts`**

Add these imports at the top of the file (after existing imports):

```typescript

```

- [ ] **Step 2: Add helper function before `CliImpl` class**

Add after `createPluginContext()` function (around line 42):

```typescript
function isBinaryPlugin(filePath: string): boolean {
  return /\.(node|dll|so|dylib)$/.test(filePath);
}

function resolvePluginPath(p: string): string {
  if (path.isAbsolute(p)) return p;
  return path.resolve(p);
}
```

- [ ] **Step 3: Modify the `run()` method to load plugins**

Replace the existing `run()` method in `CliImpl` with:

```typescript
async run(argv: string[]): Promise<void> {
  try {
    // Add dynamic namespace help text before parsing
    const namespaceHelp = this.generateNamespaceHelp();
    if (namespaceHelp) {
      this.state.program.addHelpText('after', '\n' + namespaceHelp);
    }

    // Phase 1: Pre-parse to get global options (--plugin, --input)
    // We use a temporary program clone to parse without executing commands
    const userArgs = argv.slice(2);
    const tempProgram = new Command();
    tempProgram
      .exitOverride()
      .allowUnknownOption(true)
      .addOption(new Option('-i, --input <path>', 'Override input OpenAPI path/url'))
      .addOption(new Option('-p, --plugin <paths...>', 'Extra plugin paths'));

    let globalPluginPaths: string[] | undefined;
    try {
      tempProgram.parse(userArgs, { from: 'user' });
      const tempOpts = tempProgram.opts();
      globalPluginPaths = tempOpts.plugin as string[] | undefined;
    } catch {
      // Pre-parse may throw for --help/--version; that's fine, fall through
    }

    // Phase 2: Load external JS plugins
    if (globalPluginPaths) {
      for (const pluginPath of globalPluginPaths) {
        const resolved = resolvePluginPath(pluginPath);
        try {
          if (isBinaryPlugin(resolved)) {
            console.warn(`Binary plugin skipped in JS path: ${resolved}`);
            continue;
          }
          const pluginModule = await import(url.pathToFileURL(resolved).href);
          const plugin = pluginModule.default || pluginModule;
          const plugins = Array.isArray(plugin) ? plugin : [plugin];
          for (const p of plugins) {
            this.use(p);
          }
        } catch (err) {
          const message = err instanceof Error ? err.message : String(err);
          console.warn(`Plugin load failed: ${pluginPath} — ${message}`);
        }
      }
    }

    // Phase 3: Normal command execution with loaded plugins
    await this.state.program.parseAsync(userArgs, { from: 'user' });
  } catch (error) {
    const err = error as Error & { code?: string };
    const message = err.message;

    if (message === '(outputHelp)' ||
        message === '(outputVersion)' ||
        message === '(displayHelp)' ||
        message === '(displayVersion)' ||
        message.startsWith('output') ||
        message.startsWith('display') ||
        /^\d+\.\d+\.\d+$/.test(message)) {
      return;
    }

    const errorMessage = error instanceof Error ? error.message : String(error);
    console.error(`Error: ${errorMessage}`);
    process.exitCode = 1;
  }
}
```

Note: This replaces the existing `run()` method entirely (lines 257-293). The `Option` import is already available from the `commander` import at the top.
